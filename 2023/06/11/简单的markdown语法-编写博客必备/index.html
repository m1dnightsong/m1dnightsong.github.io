先学会一些markdown语法吧

一，标题

在n级标题前加上n个“#”与1个空格。

二，字体

加粗

对象前后分别加上两个“*”。

斜体

对象前后分别加上一个“*”。

斜体加粗

那当然是三个“*”咯。

删除线

对象前后分别加上两个“~”。

三，引用

1，在行首使用符号“>”。

2，可以连用以进行嵌套引用。

四，分割线

三个及以上的“-”“*”或“_”，三者效果相同。

五，图片

1，插入线上图片

语法结构为：

！[图片描述]（图片链接’’图片标题’’）

其中“图片描述”与“图片标题”为选填。

2，插入本地图片

语法结构为：

！[图片描述]（图片路径’’图片标题’’）

同样的其中“图片描述”与“图片标题”为选填。

六，超链接

语法结构为：

[超链接名](超链接地址 "超链接标题")

其实与插入图片类似，只是无需其中的“！”

七，列表（分条列举）

1，无序列表

以“-”“+”与“*”来表示每条前的小黑点

如：

  • 纵浪大化中，不喜亦不惧。
2，嵌套

在两级间press”tab“以缩进

如：

  • 你好
    • 哟哟哟哟哟哟哦哦哦！

九，代码

0，反引号的打法

1，单行代码

对象前后分别加上一个“`”。

2，代码块

1，对象前后分别加上三个“`”且单独成行。

2，在每行代码前加上制表符（press“tab”）或四个空格。

如：

print ("人生苦短，我用python。")

十，流程图

Markdown中的流程图语法名为flow。

在绘制流程图的过程中，我们需要：

​ 1，定义元素

​ 2，连接它们

#需要在代码模式下使用。

十二，补充

本地引用–相对路径

图片除了可以放在统一的 images 文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置 _config.yml 来生成。

打开项目根目录中的 *_config.yml 文件，将 _config.yml文件中的配置项 post_asset_folder 设为 true 后，执行命令 hexo new "post_name"，在 source/*posts 中会生成文章 post_name.md 和同名文件夹 post_name。

将图片资源放在 post_name 文件夹中，文章就可以使用相对路径引用图片资源了。

但是使用这种引用方式，图片只能在文章中显示，但无法在首页中正常显示。

如果希望图片在文章和首页中同时显示，可以使用标签插件语法(推荐使用这种引用方法)。

或 CDN 引用(不推荐)

除了在本地存储图片，还可以将图片上传到一些免费的 CDN 服务中。

比如Cloudinary （梯子访问）提供的图片CDN服务，在 Cloudinary 中上传图片后，会生成对应的 url 地址，将地址直接拿来引用即可。或者上传到路过图床（不用梯子）。
